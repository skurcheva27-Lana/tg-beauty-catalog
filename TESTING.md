## 2. Как запустить локально

### Вариант A — прямо в браузере (без сервера)

1. Открыть файл `tg-app/index.html` в браузере
2. Приложение запустится с браузерным моком Telegram SDK
3. В моке имя пользователя подставляется как `«Тест»`
4. Телефон сохраняется в `localStorage` под ключом `tg_phone`
5. Внизу страницы появится кнопка-заглушка MainButton (синяя полоса)
6. Вверху появится кнопка ← BackButton

> **Ограничения мока:** HapticFeedback молчит, CloudStorage работает через localStorage,
> `tg.showAlert` → стандартный `alert()`, `openLink` → `window.open`.

### Вариант B — через локальный HTTP-сервер

```bash
cd tg-beauty-catalog/tg-app
python3 -m http.server 8080
# Открыть http://localhost:8080
```

### Вариант C — в реальном Telegram (production)

1. Открыть бота: [@anuyta_beauty_bot](https://t.me/anuyta_beauty_bot)
2. Нажать кнопку «Записаться» в меню бота
3. Откроется Mini App — полноэкранное WebView

> **Важно:** В реальном Telegram работают HapticFeedback, CloudStorage, MainButton/BackButton
> в нативном стиле. Локальный мок их не воспроизводит точно.

## 3. Архитектура навигации

### Три уровня навигации

```
ТАБ-БАР (нижнее меню)
├── [💅 Каталог]    → screen-catalog
├── [👤 О мастере]  → screen-about
├── [📸 Портфолио]  → screen-portfolio
└── [📅 Мои записи] → screen-bookings

ПОТОК ЗАПИСИ (линейный, таб-бар скрыт)
catalog → service → datetime → confirm → success
         ←BackBtn  ←BackBtn   ←BackBtn  (назад нет)
                                        MainBtn → catalog
```

### Правило таб-бара

- **Виден** на экранах: `catalog`, `about`, `portfolio`, `bookings`
- **Скрыт** на экранах: `service`, `datetime`, `confirm`, `success`
- Управляется в `app.js`: константа `TAB_SCREENS = ['catalog', 'about', 'portfolio', 'bookings']`

### Экраны (полный список)

| ID экрана      | Название         | Таб-бар | BackButton | MainButton               |
|----------------|------------------|---------|------------|--------------------------|
| catalog        | Каталог          | ✅ виден | ❌         | ❌ скрыта                |
| service        | Карточка услуги  | ❌      | ✅ → catalog| ✅ «Записаться»          |
| datetime       | Дата и время     | ❌      | ✅ → service| ✅ «Продолжить →» (disabled до выбора слота) |
| confirm        | Подтверждение    | ❌      | ✅ → datetime| ✅ «Подтвердить запись» (disabled до ввода телефона ≥11 цифр) |
| success        | Успех            | ❌      | ❌ скрыта  | ✅ «Вернуться в каталог» |
| about          | О мастере        | ✅ виден | ❌         | ❌ скрыта                |
| portfolio      | Портфолио        | ✅ виден | ❌         | ❌ скрыта                |
| bookings       | Мои записи       | ✅ виден | ❌         | ❌ скрыта                |

## 6. Как изменить данные

Все данные — только в `tg-app/js/data.js`. HTML и JS менять не нужно.

### Изменить мастера

```js
// js/data.js, строки 10–19
const MASTER = {
  name:          'Студия Ани',      // название в шапке
  masterName:    'Анна Смирнова',   // имя на экране успеха и «О мастере»
  rating:        4.9,
  reviewsCount:  127,
  description:   'Мастер маникюра и педикюра. Работаю с 2018 года.',
  workDays:      [1, 2, 3, 4, 5, 6], // 0=Вс, 1=Пн … 6=Сб
  workHours:     { start: 9, end: 20 },
};
```

### Добавить или изменить услугу

```js
// js/data.js, массив SERVICES
{
  id: 11,               // уникальный числовой ID
  category: 'nails',    // должен совпадать с id в CATEGORIES
  name: 'Новая услуга',
  short: 'Краткое описание для карточки',
  description: 'Полное описание.',
  duration: 60,         // в минутах
  price: 2000,          // в рублях, без символа ₽
  emoji: '💅',
  gradient: 'linear-gradient(135deg, #f093fb 0%, #f5576c 100%)',
  photos: ['💅', '✨', '🌸'],  // ровно 3 элемента
  reviews: [
    { author: 'Имя', text: 'Текст', date: 'вчера', rating: 5 },
  ],
},
```

### Изменить временные слоты

```js
// js/data.js, строки 189–193
const BASE_SLOTS = {
  morning:   ['09:00', '10:00', '11:00'],
  afternoon: ['13:00', '14:00', '15:00', '16:00'],
  evening:   ['18:00', '19:00', '20:00'],
};
```

## 7. После изменений — что проверить

После любых правок в `data.js`:
- Открыть каталог → убедиться, что все карточки отображаются
- Нажать на изменённую услугу → проверить данные на экране карточки
- Пройти полный поток до экрана успеха → убедиться в корректном саммари

После изменений в `app.js` или `app.css`:
- Проверить все экраны (TC-01 … TC-13)
- Особое внимание: таб-бар, BackButton, MainButton, анимации переходов

### Деплой изменений

```bash
# 1. Убедиться в наличии Vercel CLI
vercel --version

# 2. Задеплоить в production
cd /Users/svetlanakurcheva/Documents/Projecs/tg-beauty-catalog/tg-app
vercel --prod
```

При изменении `app.css` или `app.js` — **обновить cache-buster** в `index.html`:

```html
<!-- Текущие значения (строки в конце index.html) -->
<link rel="stylesheet" href="css/app.css?v=5">
<script src="js/data.js?v=5"></script>
<script src="js/app.js?v=5"></script>

<!-- После изменений увеличить цифру: v=5 → v=6 -->
```

## 8. Известные ограничения (MVP)

| Ограничение | Где | Что делать в V2 |
|-------------|-----|-----------------|
| Слоты генерируются случайно по хешу даты, не из БД | `data.js → getSlotsForDate()` | Заменить на API-запрос к бэкенду |
| Бронирование не сохраняется на сервере | `app.js → Screens.confirm` | Добавить POST-запрос к API |
| «Мои записи» показывает только текущую сессию | `app.js → Screens.bookings` | Загружать историю из API по user_id |
| Портфолио — emoji-заглушки вместо фото | `js/data.js → services[].photos` | Заменить на реальные URL фотографий |
| `initData` не верифицируется на сервере | — | Добавить верификацию по HMAC |
| Нет напоминаний о записи | — | Реализовать через бота (уведомления за 24ч и 2ч) |

## 9. Быстрая отладка

### Сбросить состояние приложения (localStorage)

```js
// Выполнить в DevTools Console
localStorage.clear();
location.reload();
```

### Посмотреть текущий state

```js
// В браузере state недоступен снаружи IIFE.
// Временно для отладки добавить в app.js:
window.__state = state;
// Затем в консоли:
__state
```

### Проверить что используется мок, а не реальный SDK

```js
// В консоли браузера:
Boolean(window.Telegram?.WebApp?.initData)
// → false — мок активен
// → true  — реальный Telegram SDK
```

### Найти экран в DOM

```js
// Все экраны всегда в DOM, активный имеет класс 'active'
document.querySelectorAll('.screen.active')
```
