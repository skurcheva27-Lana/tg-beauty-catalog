# Бриф: Telegram Mini App — Каталог услуг бьюти-мастера

> Документ разработан на основе research.md.
> Версия: 1.0 (MVP)
> Дата: март 2026

---

## Суть продукта

Клиент открывает Telegram-бота → нажимает кнопку → видит каталог услуг с фото, ценами и длительностью → нажимает «Записаться» → выбирает дату и время → подтверждает контакт → получает подтверждение в боте.

**Цель:** запись за 90 секунд без выхода из Telegram.

---

## Технический стек (рекомендация)

| Слой | Технология |
|---|---|
| Frontend | React + Vite (или SvelteKit) |
| Стили | CSS с переменными `--tg-theme-*`, никаких хардкодов |
| Маршрутизация | React Router v6 (SPA, никаких `window.location`) |
| Telegram API | `@twa-dev/sdk` или прямое `window.Telegram.WebApp` |
| Backend | Node.js / Python + bot (aiogram / telegraf) |
| База данных | PostgreSQL (записи, услуги, слоты) |
| Изображения | Cloudinary / Cloudflare Images (WebP, CDN) |
| Хостинг | HTTPS обязателен (Vercel / Railway / VPS) |

### Экран 1: Каталог услуг

**Назначение:** первый экран, точка входа, витрина всех услуг.

#### Что видит пользователь

```
┌────────────────────────────────┐
│ 💅 Студия Ани          [⭐4.9] │  ← шапка: название + рейтинг
├────────────────────────────────┤
│ [Все] [Ногти] [Брови] [Уход]  │  ← чипы-фильтры, горизонтальный скролл
├────────────────────────────────┤
│ ┌──────────────────────────┐   │
│ │  [фото 4:3]              │   │
│ │  Гелевые ногти           │   │
│ │  60 мин  ·  2 500 ₽      │   │
│ └──────────────────────────┘   │
│ ┌──────────────────────────┐   │
│ │  [фото 4:3]              │   │
│ │  Маникюр + покрытие      │   │
│ │  45 мин  ·  1 800 ₽      │   │
│ └──────────────────────────┘   │
│  ...список продолжается...     │
└────────────────────────────────┘
```

#### Элементы

| Элемент | Описание |
|---|---|
| Шапка | Название студии (или имя мастера) + рейтинг «4.9 ★» |
| Фильтр-чипы | «Все», «Ногти», «Брови/Ресницы», «Уход» — горизонтальный скролл |
| Карточка услуги | Фото (4:3, WebP), название, длительность, цена |
| Shimmer-скелетон | Показывается пока грузятся данные, 2–3 карточки-плейсхолдера |
| Пустое состояние | «Услуги скоро появятся» — если список пуст |

#### Дизайн карточки

```
background:    var(--tg-theme-section-bg-color)
border-radius: 16px
overflow:      hidden
margin:        0 16px 12px

  [фото — aspect-ratio: 4/3, width: 100%, object-fit: cover]

  padding: 12px 16px 16px

  .name      font-size: 17px, font-weight: 600, color: var(--tg-theme-text-color)
  .meta      font-size: 14px, color: var(--tg-theme-hint-color)
             «60 мин  ·  2 500 ₽»
```

#### Действия

| Действие | Результат |
|---|---|
| Тап на карточку | Переход на Экран 2 (Услуга) |
| Тап на фильтр-чип | Фильтрация списка по категории |
| Свайп вниз по списку | Скролл (не закрытие — нужна настройка `swipeBehavior`) |

#### Telegram API

```js
Telegram.WebApp.expand()           // вызвать немедленно при init
Telegram.WebApp.BackButton.hide()  // это корневой экран
Telegram.WebApp.MainButton.hide()  // нет глобального действия на этом экране
```

| Название | 22px, font-weight 700 |
| Категория | Небольшой чип-тег под названием |
| Мета-строка | Иконка часов + длительность · Иконка рубля + цена |
| Описание | До 3 строк, кнопка «Подробнее» если длиннее |
| Рейтинг + цитата | «4.9 ★ (47 отзывов)» + одна цитата с именем и датой |
| MainButton | «Записаться», всегда видна, 50px высота |

#### Действия

| Действие | Результат |
|---|---|
| BackButton (Telegram) | Назад на Экран 1 |
| Скролл галереи | Переключение фото |
| «Подробнее» | Разворачивает описание (без перехода) |
| MainButton «Записаться» | Переход на Экран 3 + `impactOccurred('medium')` |

#### Telegram API

```js
Telegram.WebApp.BackButton.show()
Telegram.WebApp.BackButton.onClick(() => navigate('/catalog'))

Telegram.WebApp.MainButton.show()
Telegram.WebApp.MainButton.setText('Записаться')
Telegram.WebApp.MainButton.onClick(() => {
  HapticFeedback.impactOccurred('medium')
  navigate('/booking/datetime')
})
```

Telegram.WebApp.MainButton.color = '#AAAAAA'  // серый пока не выбрано

// При выборе слота:
Telegram.WebApp.MainButton.isActive = true
Telegram.WebApp.MainButton.color = Telegram.WebApp.themeParams.button_color
HapticFeedback.impactOccurred('light')
```

### Экран 4: Подтверждение записи

**Назначение:** минимум данных для записи, максимум уверенности что всё правильно.

#### Что видит пользователь

```
┌────────────────────────────────┐
│ [← назад]                      │
├────────────────────────────────┤
│ Ваши данные                    │  ← заголовок секции
│ ┌──────────────────────────┐   │
│ │ Имя                      │   │
│ │ Катя Петрова             │   │  ← prefill из Telegram
│ └──────────────────────────┘   │
│ ┌──────────────────────────┐   │
│ │ Телефон                  │   │
│ │ +7 ___ ___-__-__         │   │  ← input type="tel"
│ └──────────────────────────┘   │
│ ┌──────────────────────────┐   │
│ │ Комментарий (необязательно│   │
│ │                          │   │
│ └──────────────────────────┘   │
├────────────────────────────────┤
│ Ваша запись                    │  ← заголовок сводки
│ ┌──────────────────────────┐   │
│ │ 💅 Гелевые ногти         │   │
│ │ 📅 Среда, 3 марта        │   │
│ │ 🕐 14:00 — 15:00         │   │
│ │ 💰 2 500 ₽               │   │
│ └──────────────────────────┘   │
└────────────────────────────────┘
      [Подтвердить запись]        ← MainButton
```

#### Элементы

| Элемент | Описание |
|---|---|
| Поле «Имя» | Предзаполнено из `initDataUnsafe.user.first_name + last_name` |
| Поле «Телефон» | `input type="tel"`, обязательное, маска +7 |
| Поле «Комментарий» | `textarea`, необязательное, placeholder «Пожелания, аллергии...» |
| Сводка записи | Услуга, дата, время — диапазон (старт + длительность), цена |
| MainButton | «Подтвердить запись» — активна после заполнения телефона |

#### Валидация

- Телефон: минимум 11 цифр (Россия). Пока не заполнен → MainButton disabled.
- При попытке отправить с пустым телефоном → `notificationOccurred('error')` + подсветка поля.
- На этом экране: `enableClosingConfirmation()` — защита от случайного закрытия свайпом.

  .label  color: var(--tg-theme-hint-color), font-size:14px
  .value  color: var(--tg-theme-text-color), font-size:14px, font-weight:500
```

#### Действия

| Действие | Результат |
|---|---|
| BackButton | Назад на Экран 3 (данные формы сохранены) |
| Ввод телефона | MainButton активируется |
| MainButton «Подтвердить запись» | `showProgress(true)` → API → переход на Экран 5 |
| Свайп вниз | Диалог подтверждения «Закрыть? Данные не сохранятся» |

#### Telegram API

```js
Telegram.WebApp.enableClosingConfirmation()
Telegram.WebApp.BackButton.show()

// При заполнении телефона:
Telegram.WebApp.MainButton.isActive = true

// При нажатии:
Telegram.WebApp.MainButton.showProgress(true)
Telegram.WebApp.MainButton.isActive = false
// → fetch('/api/booking', { method: 'POST', body: ... })
// → при успехе: navigate('/success')
// → при ошибке: MainButton.hideProgress(), showAlert('Ошибка записи. Попробуйте ещё раз.')
```

## Состояния и edge cases

| Ситуация | Поведение |
|---|---|
| Данные загружаются | Shimmer-скелетон вместо карточек |
| Ошибка сети | Экран ошибки с кнопкой «Повторить» |
| Нет свободных слотов на дату | Сообщение «На этот день нет свободного времени — выберите другой» |
| Слот занят в момент подтверждения | `showAlert('К сожалению, это время уже занято. Выберите другое.')` → Экран 3 |
| Пользователь без имени в Telegram | Поле «Имя» пустое, заполняется вручную |
| Телефон уже сохранён (CloudStorage) | Поле предзаполнено, пользователь может изменить |
| Нет фото у услуги | Показать цветной плейсхолдер с эмодзи категории, не пустую рамку |
