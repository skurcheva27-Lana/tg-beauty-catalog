## 5 примеров Telegram Mini Apps / ботов

---

### 1. YCLIENTS — Telegram Mini App

**Что делает:** Лидер рынка онлайн-записи в России (30 000+ салонов). Их Mini App — это белая метка: любой салон получает встроенную веб-запись внутри Telegram. Полный цикл: категория услуги → услуга → мастер → дата/время → подтверждение.

#### Сильные стороны
- Сильная синхронизация с реальным расписанием (нет двойной записи)
- Профили мастеров с фото и специализациями
- SMS + Telegram-подтверждение после записи
- Интеграция с программой лояльности
- Автоматический запрос отзыва через 2ч после визита
- Напоминания клиенту через бота

#### Слабые стороны
- «Выглядит как мобильный сайт, а не Telegram» — тяжёлый WebView, долгая загрузка
- Цена не видна сразу — нужно кликать в карточку (высокий drop-off)
- Нет фото работ мастера в потоке записи
- Мультисервисная запись (несколько услуг за раз) — неудобно
- Календарь на мобильном: сетка месяца, много тапов
- Нет ярлыка «Любимый мастер» для постоянных клиентов

#### Что мы можем сделать лучше
- Цена + длительность сразу в карточке услуги, без клика
- Фотогалерея работ как первоклассная фича
- Горизонтальный стрип недели вместо сетки месяца
- Быстрая перезапись для постоянных клиентов («Снова к Ане?»)
- Оптимистичный UI — подтверждение мгновенно, без задержки

---

### 2. Dikidi — Telegram-бот с записью

**Что делает:** Конкурент YCLIENTS, популярен у мастеров ногтей, лашмейкеров, одиночных мастеров. Их решение — разговорный бот в чате Telegram (не полноценный Mini App). В 2024 году начали пилот Mini App для отдельных партнёров.

**Поток бота:**
`/start` → Inline-кнопка «Записаться» → Категория → Услуга (цена сразу) → Мастер → Календарь → Время → Поделиться номером → Подтверждение

#### Сильные стороны
- Цена указана на шаге выбора услуги — нет сюрпризов в конце
- Кнопка «Поделиться контактом» — элегантная верификация телефона
- Подтверждение с `.ics`-файлом — можно добавить в Google/Apple Calendar
- Запрос отзыва отправляется через 2 часа после окончания записи
- Ощущение персонального общения с мастером

#### Слабые стороны
- Формат чат-бота: нет богатого UI, фото плохо вписываются
- Навигация по датам инлайн-кнопками: нудно листать на 3 недели вперёд
- Нельзя сравнить доступность нескольких мастеров
- При ошибке mid-flow — рестарт обязателен, «назад» нет
- Нет портфолио мастера или фото услуг

#### Что мы можем сделать лучше
- Mini App с визуальным портфолио, сохраняя простоту Dikidi
- Нормальная навигация назад на каждом шаге
- Одновременный выбор нескольких услуг (маникюр + педикюр)
- Визуальный выбор времени вместо текстовых кнопок

---

### 3. Altegio (ex-Reservi) — Mini App

**Что делает:** Международная ветка YCLIENTS для рынка СНГ и Европы. Более полированный Mini App с нативным ощущением Telegram.

**Отличия от YCLIENTS:**
- Карточки мастеров показывают рейтинг (звёзды) + число отзывов крупно
- В карточке услуги: «60 мин · 3 500 ₽» — длительность + цена одной строкой
- Кастомные иконки категорий вместо стоков
- Секция «Популярные услуги» в начале (на основе данных о бронированиях)
- Возвращающийся клиент: «Ваш обычный мастер: Аня» — шорткат к быстрой записи

#### Сильные стороны
- Длительность + цена вместе снимают возражение «стоит ли тратить время»
- Отсортированный по рейтингу список мастеров — социальное доказательство сразу
- Поток для постоянника сжат до 2 шагов

#### Слабые стороны
- Всё ещё медленно грузится на старых телефонах
- Фотогалереи — опциональные для владельцев; большинство не настраивает → пустые плейсхолдеры подрывают доверие

#### Что мы можем сделать лучше
- Сделать загрузку фото обязательной частью онбординга мастера (сломанный UI без фото = мотивация залить)
- Lazy loading с blur-up плейсхолдером вместо пустых рамок
- Оптимизация производительности с WebP-форматом

### 4. Боты одиночных мастеров (no-code: Salebot, BotHelp)

**Что делает:** Категория самодельных ботов, которые мастера собирают без кода через конструкторы. Типичный флоу: `/start` → список услуг с ценами → выбор времени из списка → номер телефона → подтверждение вручную мастером.

#### Сильные стороны
- Ощущение личного общения — клиент пишет напрямую мастеру
- Быстро, минимум шагов
- Мастер часто отправляет 3–5 фото работ прямо в чат перед кнопкой записи
- **Ключевой инсайт:** «Сначала работы — потом продажа» стабильно конвертирует лучше, чем сразу форма записи

#### Слабые стороны
- Нет реального календаря: мастер вручную подтверждает запись
- Нет автонапоминаний
- При занятом времени — неловкий диалог «ой, это время уже занято»
- No-code боты не масштабируются при росте

#### Что мы можем сделать лучше
- Взять «работы перед записью» как паттерн: показывать портфолио на главном экране
- Автоматический реальный календарь без ручного подтверждения
- Мгновенные напоминания

### 5. Кейсы разработчиков: Nail Studio + Barbershop Mini Apps

**Что делает:** Несколько кейс-стади опубликовано на vc.ru и Habr.com (2024) — реальные Mini Apps для студии ногтевого сервиса и барбершопа, построенные на React + Telegram Mini App SDK.

**Данные из A/B-тестов:**
- Цена сразу в карточке vs. клик для цены: **-31% drop-off** при цене сразу
- Добавление 3 фото на услугу: **+22% к записям** vs. карточка только с иконкой
- Длительность услуги в карточке: **-40% отказов** на шаге выбора времени (люди понимают, почему слоты стоят через 2 часа)

**Инсайт по потокам:**
- «Сначала мастер, потом услуга» побеждает в **барбершопе** (мужчины привязаны к конкретному мастеру)
- «Сначала услуга, потом мастер» побеждает в **ногтевых студиях и у лашмейкеров** (сервис/результат — главное решение)

#### Сильные стороны
- Нативное ощущение Telegram (используют CSS-переменные темы)
- Визуально богатые карточки с фото
- Правильная реализация BackButton и MainButton

#### Слабые стороны
- Часто нет хорошего UX для постоянных клиентов
- Поток ещё линейный: нет мультисервисной корзины

#### Что мы можем сделать лучше
- Корзина услуг (добавить маникюр + педикюр + гель за одну запись)
- Умный поток: спрашивать «сначала мастер или услуга?» — или определять по нише

## Обязательные элементы

### Telegram API (нельзя без них)

```
MainButton  — главное CTA-действие внизу экрана
             «Забронировать», «Продолжить», «Подтвердить запись»

BackButton  — навигация назад; показывать на всех экранах кроме корневых
             Скрывать на экране успеха (нельзя вернуться к форме повторно)

HapticFeedback — тактильный отклик:
  · Выбор слота         → impactOccurred('light')
  · Выбор мастера       → impactOccurred('medium')
  · Запись подтверждена → notificationOccurred('success')
  · Ошибка формы        → notificationOccurred('error')

Telegram.WebApp.expand()           — вызвать сразу при init (полный экран)
enableClosingConfirmation()        — включить на шаге 2+ формы записи
initDataUnsafe.user                — префилл имени клиента
CloudStorage                       — сохранить телефон для повторных записей
```

### UI-элементы на каждом ключевом экране

**Карточка услуги (обязательно):**
- Фото услуги / результата работы
- Название + категория
- **Цена + длительность в одной строке** («3 500 ₽ · 60 мин»)
- Краткое описание

**Карточка мастера (обязательно):**
- Фото мастера (без фото — нет записи)
- Имя, специализация
- Рейтинг в формате «4.9 ★ (127)» — всегда с числом отзывов
- Ближайшее свободное время

**Выбор даты/времени:**
- Горизонтальный стрип недели (не сетка месяца)
- Слоты сгруппированы: Утро / День / Вечер
- Серые недоступные слоты
- Подпись «Эта услуга займёт 90 мин»

**Форма подтверждения:**
- Поля: Имя (префилл из Telegram), Телефон, Комментарий
- Полная сводка записи на той же странице
- MainButton с `.showProgress()` во время отправки

**Экран успеха:**
- Анимированная галочка + `notificationOccurred('success')`
- Все детали записи
- Кнопка «Добавить в календарь» (.ics)
- BackButton скрыт

## Конкурентный gap-анализ

| Фича | YCLIENTS | Dikidi | Altegio | **Наш шанс** |
|---|---|---|---|---|
| Цена в карточке услуги | Частично (диапазоны) | Да | Да | Точная цена + длительность |
| Фотогалерея работ | Редко | Нет | Редко | **Первоклассная фича** |
| Портфолио мастера | Только аватар | Только аватар | Рейтинг | Фото + отзывы + портфолио |
| Шорткат для постоянника | Нет | Нет | Базово | Полный rebook за 2 шага |
| UX выбора даты | Сетка месяца | Инлайн-кнопки | Стрип недели | Стрип + чипы быстрых дат |
| Скорость подтверждения | 5–15 сек | 3–8 сек | 3–5 сек | Оптимистичный UI, мгновенно |
| Мультисервисная запись | Неудобно | Нет | Нет | Нативная корзина |
| Ближайшее время в карточке мастера | Нет | Нет | Да | Да + сигнал срочности |

**Главный незакрытый gap:** никто не делает фотогалереи работ хорошо внутри Telegram Mini App. Это самый высокорычажный дифференциатор для бьюти-ниши, где визуальный результат — основа решения о покупке.

## Почему клиенты уходят из воронки

По данным UX-исследований (ranked by frequency):

1. **Цена не видна до шага 4+** — пользователи, не видящие цену до 2-го шага, уходят в 3x чаще
2. **Запрос телефона слишком рано** — телефон должен быть ПОСЛЕДНИМ шагом
3. **Нет свободных слотов видно заранее** — «Ближайшее: 15 марта» на карточке мастера предотвращает потерянные усилия
4. **Медленная загрузка** — каждые +1.5 сек на переходе дают +20% отказов
5. **Запутанный календарь** — сетка месяца vs. горизонтальный стрип: стрип значительно лучше
6. **Нет кнопки «назад»** — пользователь, который не может вернуться, уходит совсем
7. **Расплывчатые названия услуг** — «Маникюр» без уточнения (гель/обычный/французский) вызывает вопрос → уход
8. **Задержка подтверждения >5 сек** — тревога «а записался ли я?» → спам «а записали ли меня?» в бота

## Рекомендованный поток нашего приложения

```
Бот → Кнопка «Открыть» → Mini App (expand())

HOME (tab): Приветствие + категории + популярные услуги
  ↓
КАТАЛОГ (tab): Фильтр по категориям → карточки [фото + цена + время]
  ↓
УСЛУГА: Фото-галерея + описание + мастера + отзывы
                                    [MainButton: Записаться]
  ↓
МАСТЕР: (если несколько) Фото + рейтинг(число) + портфолио + ближайшее время
  ↓
ДАТА/ВРЕМЯ: Стрип недели → слоты (утро/день/вечер)
                                    [MainButton: Продолжить →]
  ↓
ПОДТВЕРЖДЕНИЕ: Имя (pre-fill) + Телефон + Комментарий + Сводка
                                    [MainButton: Подтвердить запись]
  ↓
УСПЕХ: ✓ анимация + детали + [Добавить в календарь]
  [BackButton скрыт] [MainButton: На главную]

MY BOOKINGS (tab): Предстоящие + прошедшие + [Перенести] [Отменить]
PROFILE (tab): Данные + уведомления + история
```

**Для постоянного клиента (быстрый путь):**
```
HOME → «Снова к Ане на маникюр?» → Выбор даты → Подтверждение
(2 шага вместо 5)
```

## Экспертная оценка

> Три точки зрения на проект: UX-дизайнер, маркетолог, Telegram-разработчик.

---

### Взгляд UX-дизайнера

#### Типичные ошибки новичков

**1. Приветственный экран как первый экран.**
«Добро пожаловать в студию Ани!» — лишний шаг. Пользователь уже знает, куда пришёл — он нажал кнопку в боте. Первый экран должен быть каталогом, не презентацией.

**2. Слишком много категорий сразу.**
Показать 12 категорий на главной — паралич выбора. Оптимум: 4–6 категорий с иконками. Остальное — в «Ещё».

**3. Каждая услуга — отдельная страница вместо скроллируемого списка.**
Лишний переход = лишний повод уйти. Список с разворачиваемыми карточками или bottom sheet с деталями конвертирует лучше, чем полная страница на каждую позицию.

**4. Форма записи раздроблена на 5+ экранов.**
Всё, что можно объединить на одном скролле — объединяй. Выбор даты + времени — один экран. Контакты + сводка — один экран.

**5. Нет индикатора прогресса.**
«Шаг 2 из 4» — пользователь понимает, сколько осталось. Без него кажется, что конца не видно — и уходят.

**6. Кнопка «Назад» сделана вручную в шапке.**
Конфликтует с нативной BackButton Telegram. Пользователь нажимает встроенную кнопку Telegram — ничего не происходит. Это сразу ломает доверие к приложению.

**7. Нет состояний загрузки.**
Белый экран пока грузятся данные выглядит как баг. Обязательны: shimmer-скелетоны для карточек, `showProgress()` на MainButton, плейсхолдеры для изображений.

**8. Нет пустых состояний.**
«Нет записей» без объяснения и призыва к действию — тупик. Пустой экран всегда должен говорить что делать: «Вы пока ничего не бронировали — [Посмотреть услуги]».

**9. Зона касания меньше 44px.**
Кнопки и иконки мельче 44×44px — пальцем не попасть. На мобильном это критично.

**10. Не вызывается `expand()`.**
Mini App открывается в половину экрана. Пользователь видит половину каталога и думает, что это всё. Всегда `Telegram.WebApp.expand()` в первую же миллисекунду.

---

#### Что обязательно должно быть

| Элемент | Почему |
|---|---|
| Shimmer-загрузка карточек | Нет ощущения «сломалось» |
| Состояние ошибки + «Повторить» | Мобильная сеть обрывается |
| Пустые состояния с CTA | Пользователь не должен тупить |
| Disabled-стейт MainButton | Видно, что действие недоступно пока не заполнено |
| `showPopup()` для подтверждений | Не встроенный `confirm()` браузера — он выглядит уродливо |
| Отображение цены везде | Ни один экран не должен скрывать цену |

---

#### Что бесит пользователей больше всего

- **Случайное закрытие свайпом** в середине заполненной формы. Решение: `enableClosingConfirmation()` начиная со 2-го шага.
- **Нажал кнопку — ничего не произошло.** Нет визуального отклика. Пользователь нажимает ещё раз → двойная запись. Решение: сразу `showProgress()` + `isActive = false`.
- **Принудительный ввод телефона до показа услуг.** Это как в магазине попросить паспорт на входе.
- **Прокрутка конфликтует с закрытием Telegram.** На Android тянуть список вниз иногда закрывает Mini App вместо прокрутки. Решение: правильная настройка `swipeBehavior`.
- **Уведомление «Запись создана» приходит через 30 секунд.** Уже думали, что не сработало.

### Взгляд маркетолога

#### Что заставит нажать «Записаться»

**1. Визуал решает всё.**
В бьюти покупают глазами. Результат маникюра, окраски, укладки — это то, что заставляет кликнуть «Хочу так же». Фото работ — это не украшение, это главный продающий элемент. Без фото — это прайс-лист, не каталог.

**2. Социальное доказательство в нужном месте.**
Рейтинг «4.9 ★ (127 отзывов)» должен быть виден до того, как пользователь решил записываться, — прямо на карточке мастера. Одна цитата последнего отзыва под именем мастера работает лучше, чем страница из 50 отзывов.

**3. Дефицит и срочность — но честные.**
«Осталось 3 слота на эту неделю» и «Следующее свободное время: завтра в 14:00» — это не манипуляция, а полезная информация. Она снимает откладывание на потом.

**4. Снятие риска.**
«Отмена за 2 часа — без штрафа» убирает главный страх при первом бронировании. Клиент не знает мастера, не знает результат — дать возможность выйти без потерь снижает барьер.

**5. Персонализация из Telegram.**
«Привет, Катя!» на главном экране — это бесплатно (берётся из `initDataUnsafe.user`), но создаёт ощущение «меня здесь знают».

**6. Ценовое якорение.**
Если есть дорогая услуга — поставить её первой в категории. На её фоне средние по цене будут казаться выгодными. Классический приём меню в ресторане.

---

#### Элементы, повышающие конверсию

| Элемент | Эффект |
|---|---|
| Фото до/после в карточке услуги | +22% к записям (данные из исследования) |
| Точная цена без «от» | -31% drop-off |
| «Ближайшее время» на карточке мастера | Снимает неопределённость, ускоряет решение |
| Занятые слоты серым (не скрывать) | Создаёт ощущение спроса |
| Промокод для первого визита | Снижает барьер для новых клиентов |
| Rebook-шорткат для постоянников | 70–80% конверсия vs 30–40% у полного флоу |
| Напоминание о незавершённой записи | Возвращает ушедших (retargeting через бот) |
| «Добавить в календарь» после записи | Снижает no-show на 20–40% |

#### Антипаттерны, убивающие продажи

- **Диапазоны «от 2000 ₽».** Пользователь читает «от» как «будет дороже». Точная цена всегда конвертирует лучше.
- **Скрывать занятые слоты.** Выглядит как «мастер всегда свободна» — нет ощущения спроса.
- **Куча услуг без структуры.** Список из 30 позиций без категорий и фото = чувство хаоса = уход.
- **Нет возможности написать мастеру.** Иногда клиент хочет уточнить — если некуда написать, уйдёт к тому, у кого можно.

### Взгляд Telegram-разработчика

#### Ограничения, о которых нужно знать до начала разработки

**1. Хостинг полностью на твоей стороне.**
Telegram не хостит Mini App. Нужен HTTPS-сервер с валидным SSL. Все изображения — тоже на своём CDN или через Cloudflare Images/Cloudinary. Это важно: тяжёлые фото без оптимизации убьют производительность.

**2. Уведомления — только через бота, не из Mini App.**
Mini App не может отправить push. Напоминание за 2 часа до записи — это задача backend + bot.sendMessage(). Mini App только инициирует запись, бот делает всю коммуникацию.

**3. `sendData()` работает только из inline-кнопки и только строки до 4096 байт.**
Если Mini App открыта из меню бота (Menu Button) — `sendData()` не работает. Для передачи данных из Mini App на сервер лучше использовать обычный REST API с верификацией через `initData`.

**4. `initData` действительна 24 часа.**
Если пользователь не переоткрывал бота больше суток — `initData` протухла. Серверная верификация это поймает. Нужно обработать этот кейс (попросить переоткрыть бота).

**5. CloudStorage — не для чувствительных данных.**
Это client-side хранилище в Telegram. Телефон клиента хранить там можно как кеш для предзаполнения формы, но не как единственный источник истины.

**6. Telegram Payments — не везде доступны.**
Для оплаты через Telegram нужен подключённый payment provider (ЮKassa, Stripe и др.), и это требует верификации бота. Для MVP — принимать как «оплата на месте», онлайн-оплату добавить позже.

**7. openLink() открывает в браузере Telegram, не в стандартном.**
Ссылка на Google Maps откроется в урезанном браузере внутри Telegram. Для карты лучше использовать `tg://` deeplink или `openLink(url, {try_instant_view: true})`.

#### Что работает хорошо — используй это максимально

```
MainButton
  — нативный feel, всегда в правильном месте
  — setText() меняет надпись динамически: «Выбрать время» → «Подтвердить»
  — showProgress() — пользователь видит, что идёт запрос
  — setParams({color}) — можно красить под бренд (но осторожно с темой)

HapticFeedback
  — делает приложение «живым» без единой строки визуального кода
  — каждый тап на слот времени: impactOccurred('light')
  — подтверждение записи: notificationOccurred('success')

initDataUnsafe.user
  — first_name, last_name, username, language_code — бесплатно
  — префилл имени в форме, приветствие на главной

BackButton
  — нативная кнопка «назад» Telegram
  — НИКОГДА не делай свою кнопку «назад» в шапке — только через BackButton API

CloudStorage
  — сохранить телефон и имя после первой записи
  — флаг «онбординг просмотрен»
  — незавершённую запись (черновик) — пользователь вернётся и увидит

showPopup / showAlert / showConfirm
  — нативные диалоги Telegram, не browser alert
  — использовать для: подтверждения отмены, предупреждений об ошибках
```

#### Что лучше не делать

| Антипаттерн | Почему плохо | Как правильно |
|---|---|---|
| `window.location.href` для навигации | Перезагружает Mini App полностью | SPA-роутер (React Router, Vue Router) |
| `history.back()` | Может закрыть Mini App вместо перехода назад | BackButton API + собственный стек |
| `alert()` / `confirm()` браузера | Выглядит как баг вне контекста Telegram | `Telegram.WebApp.showPopup()` |
| Жёстко заданные цвета | Сломает тёмную тему и кастомные темы Telegram | Только `var(--tg-theme-*)` |
| Анимации на `transform: rotateX/Y` | Глючит в WebView на Android | Только `translateX/Y` + `opacity` |
| Fetch без обработки ошибок | Мобильная сеть обрывается — белый экран | try/catch + состояние ошибки + «Повторить» |
| Скрывать MainButton и показывать много мелких кнопок | Теряется нативное ощущение | Один MainButton + вторичные действия текстом |
| Хранить токены/секреты в CloudStorage | Client-side, небезопасно | Только на сервере, аутентификация через initData |
| Открывать Mini App из обычной URL-кнопки | Не получишь initData пользователя | Использовать именно WebApp-кнопку в боте |
| Тяжёлые изображения без оптимизации | +1.5 сек = +20% отказов | WebP, lazy loading, blur-up placeholder, CDN |

### Общие рекомендации для MVP

**Минимум, с которого стоит стартовать:**

1. **Не строй лишнего.** MVP — это каталог + запись + подтверждение. Профиль, лояльность, отзывы — следующая итерация после первых реальных пользователей.

2. **Сначала контент, потом разработка.** Приложение без фото работ — это провал на старте. Перед запуском нужно иметь минимум 3 фото на услугу и нормальное фото мастера.

3. **Один мастер проще, чем несколько.** Для V1 убери экран выбора мастера — пусть система записывает к единственному мастеру. Экран выбора мастера добавишь когда потребуется.

4. **Поток: 3 шага максимум для V1.**
   `Каталог → Выбор даты/времени → Подтверждение (имя + телефон + сводка)`

5. **Бот — основной канал коммуникации.** Всё, что происходит после записи, делает бот: подтверждение, напоминания, просьба оставить отзыв. Mini App — только точка входа и форма.

6. **Мгновенное подтверждение — не переговоры.** Запись должна создаваться сразу, без «мастер подтвердит запрос». Ручное подтверждение убивает конверсию — клиент остывает.

7. **Тестируй на реальном телефоне с Telegram, не в браузере.** Масса проблем (скролл, клавиатура, тёмная тема, haptic) видна только в настоящем Telegram на Android/iOS.

8. **Метрика успеха V1: время от открытия до подтвержденной записи.** Цель — не более 90 секунд для нового пользователя.
